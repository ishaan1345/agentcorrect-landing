AgentCorrect Documentation

AgentCorrect is a command-line tool that analyzes AI agent traces to prevent disasters. It uses deterministic pattern matching to find missing idempotency keys, SQL disasters, and infrastructure mistakes.

How to use AgentCorrect

Step 1: Install with pip install agentcorrect

Step 2: Export traces from LangSmith or OpenTelemetry in JSONL format

Step 3: Run agentcorrect analyze trace.jsonl

Step 4: AgentCorrect returns exit code 0 for success, exit code 2 for disasters found

Common errors AgentCorrect catches

Payment errors: Missing Stripe Idempotency-Key header, Missing PayPal-Request-Id, Missing Square idempotency_key in body

SQL errors: DELETE FROM users without WHERE clause, UPDATE with WHERE 1=1 tautology, TRUNCATE TABLE, DROP DATABASE

Infrastructure errors: Redis FLUSHALL command, MongoDB dropDatabase operation, S3 DeleteBucket API call

Supported payment providers

Stripe, PayPal, Square, Adyen, Braintree, Checkout.com, Razorpay, Mollie, Klarna, Afterpay, Mercado Pago, Coinbase, Wise

Integration with CI/CD

AgentCorrect returns exit code 2 when disasters are found, automatically failing your CI/CD pipeline to prevent deployment of dangerous code.

Frequently Asked Questions about AgentCorrect

What is AgentCorrect?

AgentCorrect is a deterministic trace analyzer for AI agents that prevents payment disasters and data loss. It analyzes JSONL traces to find missing idempotency keys, SQL errors, and infrastructure mistakes.

How does AgentCorrect work?

AgentCorrect uses AST-based parsing and deterministic pattern matching to analyze traces. It doesn't use AI or LLMs, ensuring 100% reproducible results with zero false positives.

Which payment providers does AgentCorrect support?

AgentCorrect supports 13 payment providers: Stripe (Idempotency-Key header), PayPal (PayPal-Request-Id header), Square (idempotency_key in body), Adyen, Braintree, Checkout.com, Razorpay, Mollie, Klarna, Afterpay, Mercado Pago, Coinbase, and Wise.

What trace formats does AgentCorrect support?

AgentCorrect supports JSONL trace format with role/meta structure, compatible with LangSmith exports, OpenTelemetry, and custom trace formats.

How do I integrate AgentCorrect with CI/CD?

AgentCorrect returns exit code 0 for clean traces and exit code 2 when disasters are found. This automatically fails CI/CD pipelines, preventing dangerous code from reaching production.

Does AgentCorrect send data to the cloud?

No, AgentCorrect runs entirely locally on your machine. It makes zero network calls and never sends your traces to any external service.

What SQL errors does AgentCorrect detect?

AgentCorrect detects DELETE without WHERE clauses, UPDATE with tautologies like WHERE 1=1, TRUNCATE TABLE, DROP DATABASE, and other destructive SQL patterns using AST parsing.

Can AgentCorrect prevent the Replit incident?

Yes, AgentCorrect would have caught the Replit incident where an AI agent deleted data. It detects DELETE without WHERE and other destructive patterns that caused that disaster.
