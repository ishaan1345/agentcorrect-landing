How It Works

1

AI agent runs in staging

Your agent makes API calls, runs SQL, modifies infrastructure

2

Actions logged to trace.jsonl

Standard JSONL format, works with any observability tool

3

AgentCorrect analyzes traces

Detects missing idempotency, SQL disasters, cache wipes

4

Exit code 2 = deploy blocked

Non-zero exit fails CI/CD. Production stays safe.

Get Started in 2 Minutes

1. Install AgentCorrect

pip install agentcorrect

2. Test with built-in demo

# See it catch real disasters
agentcorrect demo --scenario all

# Output:
[FAIL] Missing payment idempotency
   Provider: Stripe
   Fix: Add header 'Idempotency-Key: <unique-id>'

[FAIL] Dangerous SQL operation
   Query: DELETE FROM users WHERE 1=1

Exit code: 2 (blocks CI/CD)

3. Add to your CI/CD pipeline

# GitHub Actions
- name: Check agent safety
  run: |
    pip install agentcorrect
    agentcorrect analyze staging_traces.jsonl
    # Exit code 2 will fail the build

4. Optional: Generate OPA policies

# Generate enforcement policies
agentcorrect analyze traces.jsonl --out reports/ --rego

# Creates:
# - policy.rego (OPA runtime enforcement)
# - report.html (visual analysis)
# - findings.json (machine-readable)
