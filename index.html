Trace analysis for AI agents in production

You're shipping AI agents to production. You're logging traces with LangSmith. But you're not actually reading those traces to catch the disasters hiding in them.

The problem with AI agents calling APIs

AI agents don't understand API semantics. They make syntactically correct calls that are semantically wrong. They'll retry payment calls without idempotency keys. They'll execute DELETE FROM users without a WHERE clause. They work in development, then destroy production.

You can trace every call with LangSmith or LangFuse. But traces are just data. Someone needs to actually analyze them for patterns that cause production failures.

# What your agent generates (looks fine, right?)
{ "role": "http", "meta": { "http": { "method": "POST", "url": "https://api.stripe.com/v1/charges", "headers": {/* Idempotency-Key missing */}, "body": {"amount": 5000, "currency": "usd"} } } }

AgentCorrect reads your traces and finds disasters

$ pip install agentcorrect

Export your traces from LangSmith, LangFuse, or OpenTelemetry. Run the analysis:

$ agentcorrect analyze trace.jsonl Analyzing 1,247 events... SEV0: Payment call missing idempotency protection Provider: Stripe (POST /v1/charges) Line: 142 Impact: Will charge customer multiple times on retry Fix: Add 'Idempotency-Key' header with unique value SEV0: SQL DELETE without WHERE clause Query: DELETE FROM users Line: 89 Impact: Will delete all records in table Fix: Add WHERE clause or use TRUNCATE if intentional Exit code: 2 # Blocks CI/CD deployment

Add to your CI/CD pipeline. If disasters are found, deployment stops.

What it actually does

  • Payment APIs: Checks Stripe, PayPal, Square, Adyen calls for missing idempotency keys
  • SQL queries: Detects unbounded DELETEs, UPDATEs, and destructive operations
  • Cache/DB ops: Catches Redis FLUSHALL, MongoDB dropDatabase, S3 DeleteBucket
  • Output format: Human-readable reports or OPA Rego policies for runtime enforcement
  • Privacy: Runs locally, no data leaves your machine

Works with your observability stack

• Export traces from LangSmith, LangFuse, or OpenTelemetry

• Accepts any JSONL format with role and meta fields

• One-line integration with GitHub Actions, GitLab CI, Jenkins

• Standard exit codes (0 = safe, 2 = disasters found)

Why I built this

I was debugging payment failures in our AI agent. The agent code looked perfect. The LangSmith traces showed successful API calls. But customers were getting charged multiple times. Turns out the agent didn't know about idempotency keys - a concept that doesn't exist in its training data. AgentCorrect is basically a linter for AI agent traces, checking for patterns that cause production disasters.
