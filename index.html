AgentCorrect is a deterministic "effect guardrail" for agents: it analyzes your test/staging traces in CI and fails the build on catastrophic actions.

Run your agent tests

→ they write traces.jsonl

→ Sev-0 finding ⇒ exit 2 ⇒ PR fails

What we catch today

(v0, zero-FP by design)

💳

Payments

Stripe/PayPal/Square/Adyen POST without required idempotency

Why: Retries create duplicate charges
Fix: Add provider's idempotency header
Docs: Vendor-documented requirement
🗑️

SQL

DELETE/UPDATE without WHERE or tautologies (WHERE 1=1)

Why: Affects all rows, data loss
Fix: Add specific WHERE conditions
Docs: Universally catastrophic
💥

Infra nukes

Redis FLUSHALL, Mongo dropDatabase, S3 DeleteBucket

Why: Service outage, total data loss
Fix: Use targeted operations
Docs: Documented disasters

Why these? They're vendor-documented or universally catastrophic. No opinions, no ML "scores."
