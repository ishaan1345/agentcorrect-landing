Built for anyone building AI agents

Your AI agent is charging customers 5 times

I built this because I was frustrated going through traces for AI agents that dealt with payments. AgentCorrect catches duplicate charges and SQL disasters before they hit production.

$ git clone github.com/ishaan1345/agentcorrect
Works with LangSmith, LangFuse, OpenTelemetry, or any JSONL trace

What actually happens

Your AI agent retries on network timeout. Customer gets charged 3 times.

❌ Without AgentCorrect
# Your AI agent's payment code
stripe.charges.create(
amount=9999,
currency="usd",
customer=customer_id
) # No retry protection!
 
# What happens in production:
✓ Payment processed: $99.99
⚠ Network timeout, retrying...
✓ Payment processed: $99.99
⚠ Connection lost, retrying...
✓ Payment processed: $99.99
 
Customer charged: $299.97
Support ticket: "WHY 3 CHARGES??"
✓ With AgentCorrect
# Run AgentCorrect in CI/CD
$ agentcorrect analyze trace.jsonl
 
CRITICAL: Payment retry protection missing
Line 142: stripe.charges.create()
Problem: Will charge multiple times on retry
Fix: Add unique ID to prevent duplicate charges
 
Build failed - Exit code: 2
 
# After fixing:
✓ All payment calls protected
✓ SQL queries safe
✓ Safe to deploy
$ agentcorrect analyze agent_trace.jsonl
# Scanning 1,247 events from your AI agent...
 
CRITICAL ISSUES FOUND:
 
1. Payment calls missing retry protection (3 found)
→ Stripe charges at lines 142, 367, 891
→ Will cause duplicate charges on network retry
 
2. SQL DELETE missing safety checks (1 found)
→ Line 234: DELETE FROM users
→ Will delete ALL users (no WHERE clause)
 
Exit code: 2 (CI/CD pipeline blocked)
