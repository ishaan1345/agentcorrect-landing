Your AI agent just charged someone's card 47 times

Last Thursday at 3am, I got woken up by my phone buzzing non-stop. My AI agent had gone rogue.

It was supposed to process one payment. Instead, every time the Stripe API timed out, my agent retried. And retried. And retried.

support-tickets-urgent.txt
[03:42] WHY WAS I CHARGED 47 TIMES?????
[03:43] I NEED A REFUND NOW
[03:45] IS THIS A SCAM???
[03:47] I'm calling my bank
[03:51] Chargeback initiated

The problem wasn't the code. The problem was that AI agents don't know what they don't know. They don't know that Stripe needs an idempotency key. They don't know that DELETE without WHERE deletes everything. They just... do things.

So I built AgentCorrect. It reads the traces from your AI agent and catches these disasters before they happen. Before you get that 3am call.

Here's what I do now

Every time my agent runs, I save its trace. Every time before I deploy, AgentCorrect scans it. If it finds something that will cause a disaster, it stops the deployment.

$ pip install agentcorrect
$ agentcorrect analyze trace.jsonl
# Reading 1,247 events from your agent run...
 
FOUND: Payment call that will retry and charge multiple times
Line 142: POST to stripe.com/v1/charges
Missing: Idempotency-Key header
Impact: Customer will be charged on every retry
 
FOUND: SQL query that will delete all records
Line 89: DELETE FROM users
Missing: WHERE clause
Impact: Will delete entire users table
 
Exit code: 2
# Deployment blocked. Fix these first.

That's it. It reads the traces so I don't have to. It knows what patterns cause disasters because I've catalogued them all. Stripe, PayPal, Square, Adyen - each has specific requirements. SQL, Redis, MongoDB - each has dangerous operations.

The boring technical details

1

It runs locally

Your traces never leave your machine. No cloud, no uploads, no tracking.

2

It's deterministic

No AI, no ML, no hallucinations. Just pattern matching against known disasters.

3

It works with your stack

LangSmith, LangFuse, OpenTelemetry, or just JSONL files. One line in your CI/CD.

4

It outputs rules

Generates OPA Rego policies if you want to enforce rules at runtime too.

Add it to your pipeline

# .github/workflows/test.yml
 
- name: Test agent safety
run: |
python run_agent.py --save-trace trace.jsonl
agentcorrect analyze trace.jsonl

If AgentCorrect finds a disaster, it returns exit code 2. Your build fails. You fix the issue. Then you ship.
