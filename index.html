How it actually works

# Your AI agent makes API calls, logged to trace.jsonl:
{"role":"http","meta":{"http":{"method":"POST","url":"https://api.stripe.com/v1/charges","headers":{},"body":{"amount":9999}}}}
{"role":"sql","meta":{"sql":{"query":"DELETE FROM users WHERE active=false"}}}

# AgentCorrect scans for disasters:
$ agentcorrect analyze trace.jsonl --out reports/

# Generates 8 files including OPA policy:
reports/
├── policy.rego        # Runtime enforcement rules
├── findings.json      # Machine-readable violations
├── report.html        # Visual report
└── coverage.json      # Shows 95% of ops checked

# Exit codes tell CI/CD what to do:
Exit 0 = Deploy it
Exit 2 = Block deploy (disaster detected)

The tech that makes it reliable

  • • eTLD+1 extraction: Can't spoof with pay.stripe.evil.com
  • • SQL comment stripping: Can't hide tautologies in /* comments */
  • • Stateful tracking: Catches same key, different amounts
  • • Zero dependencies: Pure Python, no network calls
  • • Fast: <100ms per trace
